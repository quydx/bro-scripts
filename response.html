Gameover P2P

Names: Gameover Zeus, GOZ
DGA usage period: 2011 - 2014
Credits: Daniel Plohmann
Description: Gameover P2P was an exceptionally robust botnet active for about 2.5 years until its takedown in Operation Tovar in June 2014.
Its primary characteristic was the sophisticated P2P-based C&C protocol and it is said to be a further development of the file infecting Zeus variant LICAT / Murofret. It also used a DGA as backup.
The DGA calculated 1000 domains based on time. The validity interval start was "weeks" with respect to the day of month, thus each interval starting on the 1st, 8th, 15th, 22nd, and 29th and the domains being valid for usually 7 days if it was not the end of the month.
Characteristic for this DGA the uneven usage of TLDs, with more *.com domains being generated than the others (*.ru, *.info, *.biz, *.net, *.org). The average the domain length without TLD is around 25 characters.
Regex: [a-z]{11,32}\.(net|biz|org|com|info|ru)$
Examples: qguovypzsghmtijhljtsnflnnft.ru
myduodyxtemrcpfproteimzce.com
wsskljhihuamxcqswgkrrwdxk.net

Description: Symmi has been first described in this blog post focusing on its DGA by Johannes Bader. The earliest samples to have exposed similar DGA behaviour are dated mid December 2014.
A whole set of parameters is used to control the algorithm: Min/max subdomain length, number of domains per time period and length of this period plus a magic to shift the date-based seed. Regex: [aeioubcdfghklmnpqrstvwxz]{8,15}\.ddns.net$

SuppoBox

DGA usage period: 2013 - ?
Credits: Implementation: Johannes Bader

Description: SuppoBox has been first observed in Feburary 2013 and exposes information stealing behaviour. Especially, it self-propagates by scraping email addresses from infected systems and mailing itself to them, as described by Jason Geffner in this report.
It uses the DGA as primary C&C channel.
Variants using two different wordlists have been identified.
The DGA is based on time and will generate a new domain composed of two words from the wordlist every 512 seconds. Altogether, it will cycle through a total of 32.768 domains, always using the *.net TLD.
Regex: [a-z]{7,30}\.net$
Examples: perhapsmeasure.net
windowafraid.net
wintermeasure.net

TinyBanker DGA

Names: Tinba, Zusy
DGA usage period: 2014 - ?
Credits: Daniel Plohmann
Description: TinyBanker is an extremely compact banking malware that uses hooking to intercept and manipulate HTTP(S) requests. The name results from its small memory footprint. Since mid 2014, a new variant surfaced that is using a DGA as backup in case the primary C&C domain is unreachable. This is also one of the first consequences from the source code leak in 2014
The domains are not tied to temporal validity. A set of up to 1000 ordered names is generated. The names depend on the embedded RC4 key, the initial hardcoded domain and a single hardcoded TLD suffix.
Characteristic for this DGA is the independence from time for generation, a fixed length of 12 characters for domain names and exclusive used of the TLD harcoded in the binary. Dependence on the initial domain and the RC4 key means a unique DGA set for each campaign where this malware is used.
Regex: [a-y]{12}\..{2,6}$
Examples: ghefvfkkxtgg.ru
mqsqytogddne.ru
hosgnecdevwt.ru
